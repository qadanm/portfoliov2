## The setting

Worked on advisor-facing financial platforms built using InvestCloud's proprietary system, supporting deployments across multiple wealth management institutions.

These were multi-tenant platforms where each implementation had to be adapted to the specific branding, structure, and requirements of the institution, while still operating within a shared underlying system.

## Where I worked

My work focused on the frontend layer within InvestCloud's CMS and UI framework, where I:

- Built and configured UI components across advisor-facing interfaces
- Customized platforms to match each institution's branding and design requirements
- Implemented structured layouts for dense financial data
- Refined interaction flows across advisor tools and workflows
- Ensured consistency across screens and user states
- Supported responsive behavior across desktop, tablet, and mobile

I worked closely with design, product, and QA teams to translate requirements into functional interfaces, troubleshoot issues, and keep implementations aligned with both business needs and system constraints.

## What the platforms supported

- Portfolio management
- Advisor–client interaction
- Onboarding and lifecycle workflows
- Multi-account and multi-role environments

## The balance

The work required balancing flexibility with consistency. Each portal had to be adapted to its client while staying usable across complex, data-heavy systems.

This experience built a strong foundation in designing and implementing structured interfaces within large, shared systems.
